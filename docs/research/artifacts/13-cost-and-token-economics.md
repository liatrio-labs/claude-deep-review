# The real cost of multi-agent AI code review

**Running a multi-agent AI code review system in production costs between $0.05 and $25 per pull request**, a 500x range driven almost entirely by architecture choices, model selection, and optimization strategies. Anthropic's managed Claude Code Review averages **$15–$25 per PR** using parallel specialized agents, while self-hosted systems using prompt caching and model routing can achieve comparable review depth for **$0.50–$1.50 per PR**. The critical finding across the research: token duplication rates of **53–86%** across multi-agent frameworks mean most systems waste the majority of their compute re-processing identical context, and the quality gains from additional agents plateau sharply after **four agents** according to Google DeepMind's systematic evaluation of 180 configurations.

These numbers matter because AI-assisted code generation now produces **41% of new commits** industry-wide, while the Faros AI Productivity Paradox study found that AI adoption increases PR review time by **91%**. The bottleneck has shifted decisively from writing code to reviewing it, making the economics of automated review a first-order concern for engineering organizations.

---

## What production systems actually cost to run

The pricing landscape reveals a striking divergence between managed services and self-hosted approaches. **Anthropic's Claude Code Review**, launched in March 2026, bills per-review based on token consumption. The company's stated average of **$15–$25 per review** reflects a multi-agent architecture that dispatches parallel specialized agents — logic error detection, security vulnerability analysis, performance regression checks, and test coverage review — followed by an aggregator that deduplicates and ranks findings. Reviews take approximately **20 minutes** to complete. For a team generating 50 PRs per day, this translates to **$16,500–$27,500 per month**.

The flat-rate competitors operate on radically different economics. **CodeRabbit** charges **$24 per developer per month** for unlimited reviews and runs on **200+ Google Cloud Run instances** with 8 vCPUs and 32 GiB RAM each during peak hours. At Cloud Run's pricing of roughly $0.50–$0.80 per instance-hour, their compute infrastructure alone costs thousands monthly before LLM API calls. Their margin depends on the average per-review cost staying well below the per-developer subscription revenue — likely **$0.50–$3.00 per review** based on their scale of 13M+ PRs processed across 2M+ repositories. **Qodo** (formerly CodiumAI) charges **$30 per user per month** and triggers **15+ review agents** per Git workflow, though its open-source PR-Agent uses a single LLM call per tool at roughly **$0.02–$0.10 per review**. **Greptile** sits at **$30 per developer per month** with 50 reviews included, then **$1 per additional review**.

The most granular cost data comes from the **Kilo Code Reviewer benchmark**, which tested real PRs from the Hono framework. A 338-line PR cost **$0.73** with Claude Opus 4.6 (consuming 618,853 input tokens) versus **$0.07** with Kimi K2.5 (359,556 input tokens). A 598-line PR jumped to **$1.34** with Opus — an 83% increase for 77% more lines, suggesting mildly superlinear cost scaling. The **10x cost difference** between frontier and budget models dominates all other variables.

**GitLab** has entered aggressively at **$0.25 flat per review**, while GitHub Copilot Code Review consumes one "premium request" per PR, costing approximately **$0.04** once the monthly quota is exceeded. These represent lighter-weight, non-multi-agent approaches that sacrifice depth for affordability.

## Token duplication is the hidden cost multiplier

The most important — and least discussed — cost driver in multi-agent code review is **redundant token consumption**. The AgentTaxo study, presented at the ICLR 2025 Workshop, measured duplication rates across leading frameworks: **CAMEL at 86%**, **MetaGPT at 72%**, and **AgentVerse at 53%**. A token is counted as duplicate if it appears in at least two LLM calls. For code review, where multiple agents examine the same diff and surrounding code, these rates directly translate to wasted spend.

The numbers compound quickly. A Capgemini analysis found that a simple task requiring 28 tokens with a single agent consumed **1,158 tokens across a 5-agent system** — a **41x increase**. With three discussion rounds among five agents, round three alone costs 585 tokens just for prompt assembly. The Google DeepMind paper "Towards a Science of Scaling Agent Systems" confirmed that **turn count follows power-law scaling** with agent count, and unstructured multi-agent networks amplify errors up to **17.2x** versus single-agent baselines. Centralized coordination contains this to 4.4x, but the overhead remains substantial.

The **input-to-output token ratio** in code review is heavily skewed. AgentTaxo measured **2:1 to 3:1** across frameworks generally, but code review is even more input-heavy — typically **5:1 to 10:1** — because a large diff plus surrounding context produces relatively concise review comments. A typical single-agent code review call breaks down approximately as: **20–40% diff content**, **25–40% surrounding/referenced code**, **10–15% system prompt and instructions**, and **5–10% tool definitions**. In multi-turn agent conversations, prior conversation history can consume an additional **10–25%**.

**PR size drives cost superlinearly.** The diff itself scales linearly (roughly 10 tokens per line of code), but larger PRs touch more files and require more cross-file dependency analysis. At the **500-line threshold**, tools like CodeRabbit and monday.com report diminishing returns in review quality. At **1,000+ lines**, Anthropic's data shows **84% of PRs receive findings** averaging 7.5 issues — the review is working harder but also consuming far more context. Google's research confirms that human review quality degrades significantly above **200 lines of changed code**, and the same attention-degradation dynamics apply to LLMs processing bloated contexts.

## Six optimization strategies and their measured impact

**Prompt caching delivers the largest single savings.** Anthropic's prompt caching charges **10% of the base input price** for cached token reads (a 90% discount). For code review, where system prompts, tool definitions, coding standards, and repository context remain constant across reviews, roughly **70–80% of input tokens are cacheable**. Greptile v3 achieved **75% lower inference costs** despite using **3x more context tokens** than v2, entirely through high cache hit rates. A CostLens analysis of a code review bot showed **64% savings** at 100 requests. One developer reported monthly costs dropping from **$720 to $72** by caching 81,000 tokens of persistent context. Combined with Anthropic's batch API (50% discount for 24-hour turnaround), the stacked discount can reach **95% off standard input pricing** — reducing Sonnet 4.6's $3/MTok input to an effective **$0.15/MTok**.

**Model routing cuts costs 37–75% with minimal quality loss.** Current Claude pricing spans a 5x range: Haiku 4.5 at **$1/$5** per million input/output tokens, Sonnet 4.6 at **$3/$15**, and Opus 4.6 at **$5/$25**. A tiered architecture routing 65% of work to Haiku (triage, classification), 25% to Sonnet (standard analysis), and 10% to Opus (complex architectural reasoning) yields a weighted input cost of **$1.80/MTok versus $5.00** for all-Opus — a 64% reduction. Amazon Bedrock's Intelligent Prompt Routing reports **up to 30%** automated cost reduction, and a Bifrost case study documented monthly spend falling from **$42,000 to $18,000** through tiered routing. MT-Bench data confirms that over **50% of queries** can be effectively handled by Haiku-class models with accuracy scores above 8/10.

**Context-pulling versus context-pushing** represents an architectural choice with both quality and cost implications. cubic's transition from context-pushing (stuffing all potentially relevant codebase context upfront) to context-pulling (agents dynamically retrieve context as needed) produced **51% fewer false positives** without sacrificing recall. The token reduction is inherent: only relevant context is retrieved rather than everything. Claude Code uses this approach natively — maintaining lightweight file path identifiers and using grep/glob for dynamic retrieval, consuming **5.5x fewer tokens than Cursor** for equivalent tasks. The tradeoff is latency: cubic's agentic approach takes **2–5 minutes per review** versus seconds for simpler tools.

**Early termination** offers proportional savings. Anthropic's data shows only **31% of small PRs** (under 50 lines) receive any findings, versus **84% of large PRs** (1,000+ lines). CodeRabbit's open-source ai-pr-reviewer includes built-in "smart review skipping" for trivial changes. If 40% of PRs skip full multi-agent review after a cheap Haiku triage call (~$0.01–$0.10), the system saves roughly **40% of total cost**. The risk is real but manageable — Anthropic notes that even one-line changes can break authentication.

**Cross-agent caching** through frameworks like CrewAI prevents redundant tool calls when multiple agents query the same files or functions. CrewAI caches tool results by default and provides a unified memory system across agents. No hard percentage savings data exists, but the AgentPrune research demonstrates that **72.8% of tokens can be pruned** from multi-agent communication graphs while maintaining or slightly improving performance (+0.84%), saving up to **$177.58 per experiment** with 5 GPT-4 agents. AgentDropout (ACL 2025) achieves **21.6% prompt token reduction** with performance improvements of 1.14, suggesting that much inter-agent communication is noise.

## The quality curve bends sharply after four agents

The cost-quality tradeoff has a definite knee. Google DeepMind's systematic evaluation found that **performance plateaus at approximately four agents**, after which coordination overhead consumes marginal benefits. More strikingly, when single-agent baselines already exceed **~45% accuracy**, multi-agent coordination yields **negative returns** (β=−0.404, p<0.001). Cursor's BugBot confirmed this empirically: running 8 parallel review passes with majority voting, the system **plateaued at 35 out of 38 planted bugs after 4 passes**. The remaining 3 bugs were never caught regardless of additional passes.

The false-negative-to-false-positive cost asymmetry strongly favors erring toward recall. IBM's widely cited data (directionally valid if not precisely quantified) estimates bugs cost **$80 to fix during development** versus **$7,600 after release** — a **~100x multiplier**. NIST puts production defect costs at **$10,000+** when customer impact is included. By contrast, a false positive costs a developer **5–10 minutes** to evaluate, worth **$6–17** at typical loaded rates. This roughly **500:1 cost asymmetry** between missed bugs and false alarms means review systems should tolerate significant noise before pulling back — but the practical limit is tool abandonment. As one practitioner noted, "a tool that catches 70% of bugs but generates 20 false positives per PR will get disabled within a month."

Current AI code review quality sits in a middle ground. The Martian Code Review Bench (evaluating ~300K PRs) found **CodeRabbit at 51.2% F1** (highest overall), with most tools optimizing precision at the expense of recall. A practitioner test with 23 planted bugs found **GitHub Copilot catching 70%**, **CodeRabbit and Claude at 65%**, with no single tool catching everything — but the union of all tools reached **91%**. This argues for multi-tool or multi-agent approaches, but with diminishing returns per additional agent.

**Human code review, by comparison, costs $75–$150 per PR** at typical US senior developer loaded rates of $75–100/hour and 1–2 hours per thorough review. Google's study of 9 million reviewed changes found median review latency of **under 4 hours**, with 80% of reviews requiring developer action. The crossover point is unambiguous: even Anthropic's premium $15–$25 AI review is cheaper than a single hour of senior developer time, and budget approaches at $0.50–$1.50 per PR deliver a **50–300x cost advantage** for first-pass review.

The Faros AI Productivity Paradox adds a crucial nuance. Their telemetry from **10,000+ developers across 1,255 teams** found that AI-adopting developers complete **21% more tasks** and merge **98% more PRs**, but PR review time increases **91%** and bugs per developer rise **9%**. At the organizational level, there is **no significant correlation** between AI adoption and improvements in throughput, DORA metrics, or quality KPIs. The bottleneck has shifted downstream — Amdahl's Law in action — making AI code review arguably **more important than AI coding tools** for organizational ROI.

## What a 50 PR/day system costs today and tomorrow

For a team generating 50 PRs per day (1,100 per month), the monthly cost varies dramatically by architecture:

| Configuration | Monthly cost | Per PR |
|---|---|---|
| Claude Code Review (managed) | $16,500–$27,500 | $15–$25 |
| Self-hosted, Opus 4.6, no optimization | $2,600 | $2.36 |
| Self-hosted, Sonnet 4.6, with caching | $1,000–$1,200 | $0.91–$1.09 |
| Self-hosted, mixed routing + caching | $600–$900 | $0.55–$0.82 |
| Self-hosted, Haiku 4.5, with caching | $350 | $0.32 |
| CodeRabbit Pro (50-person team) | $1,200 | $1.09 |

These estimates assume a 5–7 agent system with a PR mix of 40% small, 40% medium, and 20% large, consuming roughly **75K–700K total tokens per review** depending on PR size and agent count. **API costs constitute 75–85% of total spend.** Infrastructure — preprocessing compute, vector database storage for embeddings, network for git operations, orchestration, and monitoring — adds approximately **$140–$410 per month**, a modest fraction dominated by the LLM tax.

The managed-versus-self-hosted gap is striking: Anthropic's **10–20x markup** over raw API costs reflects orchestration complexity, multi-agent coordination engineering, verification layers, and margin. For organizations with the engineering capacity to build their own orchestration, the savings are substantial.

**Model costs are falling at 10–50x per year** for equivalent quality. Epoch AI's rigorous analysis found a median decline of **~50x per year** across benchmarks (2020–2025), with post-January-2024 rates accelerating to **~200x per year**. GPT-4's input pricing dropped from **$30 to $2.50 per million tokens** (92%) in 18 months. Claude Opus fell from **$15/$75 to $5/$25** per million input/output tokens between March 2024 and late 2025. Projecting conservatively at 10x per year, a Sonnet-equivalent model should cost **~$0.30/$1.50 per million tokens by 2027**, bringing the 50 PR/day multi-agent system down to **$100–$200 per month** — essentially negligible against developer salary costs.

As costs drop, the binding constraint shifts from budget to **latency**. Claude Code Review already takes ~20 minutes per PR. More agents or deeper analysis extends this. The future optimization path runs through faster models, better parallelization, and smarter context retrieval — not simply cheaper tokens. The architecture that makes economic sense also evolves: when Sonnet-class quality costs $0.30/MTok, running 12 specialized agents with full repository context becomes viable for less than today's 5-agent cost, enabling review of performance, accessibility, API compatibility, test adequacy, and documentation consistency in a single automated pass.

## Conclusion

The economics of multi-agent AI code review are dominated by three variables: **model selection** (10x cost range between frontier and budget models), **token duplication** (53–86% of tokens wasted across frameworks), and **caching effectiveness** (60–90% savings on the 70–80% of tokens that are cacheable). Organizations building these systems should architect for prompt caching first, model routing second, and context-pulling third — this combination can reduce costs by **80–95%** versus naive implementations.

The quality ceiling at approximately four agents means that the marginal dollar spent on agent #5 through #9 buys increasingly less bug detection. The strongest architecture combines a small number of specialized parallel agents with aggressive deduplication and a verification pass, rather than maximizing agent count. At current pricing, a well-optimized self-hosted system delivers comparable depth to Anthropic's managed service at **$0.50–$1.50 per review versus $15–$25** — but requires significant engineering investment in orchestration, caching, and reliability.

The most underappreciated insight: the Faros AI data shows that AI code review tools may deliver **higher organizational ROI than AI coding assistants**, because they address the bottleneck (review) rather than accelerating the non-bottleneck (code generation). With model costs halving every few months, the question is no longer whether to automate code review, but how aggressively to invest in review infrastructure before the cost advantage becomes table stakes across the industry.
