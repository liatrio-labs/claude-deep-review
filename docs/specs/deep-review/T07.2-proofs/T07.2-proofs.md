## Implementation Summary

Modified `scripts/filter_findings.py`:

### parse_review_md() changes
- Added `dimension_thresholds: {}` to default config
- Added section-based parser for `## Confidence Threshold` that handles:
  - Plain integer lines (sets global default, preserves backward compat)
  - Key:value pairs like `bugs: 75` or `security: 70` (stored in `dimension_thresholds`)
  - Mixed format: plain number sets global, key:value overrides individual dims
  - HTML comment stripping so spec-style inline comments don't corrupt parsing
- When `security` appears in dimension_thresholds, updates `security_min_confidence` accordingly
- Added section-based parser for `## Severity Threshold` (plain value like `medium`)

### apply_threshold_filter() changes
- Extracts `dim_thresholds` and `security_min` from config upfront
- Per-finding threshold resolution:
  1. Start with global default
  2. For each finding dimension, if a per-dimension entry exists, take the minimum (most permissive)
  3. For security dimensions, apply `max(effective, security_min)` to enforce the 70 floor
