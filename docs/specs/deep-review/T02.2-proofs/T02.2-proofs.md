## Changes Made

Replaced the `classify_blame()` stub in `scripts/verify_findings.py` with a full implementation that:

1. Classifies cross-file impact findings (non-empty `cross_file_refs`) as "surfaced" immediately without running git blame.
2. Returns "new" (conservative) when the target file does not exist on disk.
3. Fetches PR commits via `git log --format=%H {base_branch}..HEAD`.
4. Runs `git blame -L{line_start},{line_end} -- {file}` on the finding's line range.
5. Parses blame output to extract commit SHAs using a regex that handles both full and short SHAs.
6. Classifies as "new" if any blamed SHA matches a PR commit; "surfaced" otherwise.
7. Handles edge cases: binary files, git blame failure, unparseable blame output — all conservatively classified as "new" with a warning.
8. Downgrades `severity` by one level for "surfaced" findings: critical->high, high->medium, medium->low, low stays low.
9. Records `blame_metadata` dict on the finding: `{classification, author, date, original_severity}`.
