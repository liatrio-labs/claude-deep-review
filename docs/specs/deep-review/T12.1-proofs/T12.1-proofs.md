## Changes Made

### Interactive Finding Walkthrough (replaced)

The old "Interactive Finding Walkthrough" section was a single-step flow with 4 options (no description fields) and a conditional fifth "Done" option only at severity group transitions.

The new section has two explicit steps:

**Step 1: Show Summary Table**
- Before any AskUserQuestion, output a markdown table of ALL findings
- Columns: # | Severity | Title | Confidence | File
- Grouped by severity (Critical → High → Medium → Low)
- Uses finding IDs (F-01, F-02) that match the report

**Step 2: Walk Through Each Severity Group**
- AskUserQuestion per finding with exactly 4 options, all with description fields
- Option 1: "Include as PR comment" — post as inline PR comment
- Option 2: "Skip this finding" — exclude, add to dismissed_set
- Option 3: "Include all remaining {Severity}" — bulk include, advance to next group
- Option 4: "Done — keep what I've selected" — early exit, deliver chosen findings
- Skipped findings tracked in dismissed_set

### Stage 2: Task Board (updated)

- Finding IDs (F-01, F-02) now appear in option descriptions for consistency with summary table
- Added paragraph: when walking through findings for task board, reuse the summary table already shown and reference findings by ID

### Stage 3: Dismissed Findings (updated)

- Added gate condition: only run if dismissed_set is non-empty
- Added: pre-populate proposed entries from dismissed_set (user doesn't re-identify what they already skipped)
