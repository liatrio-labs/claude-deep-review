Evidence (line 158):
  **Critical framing rule:** The summary must describe what the PR says it does -- not validate whether it succeeded.
  Frame all statements as claims: "The PR claims to reorganize X by extracting from A into B. The following functions
  were moved: [list]. Verify that the extracted code preserves the original behavior."

Evidence (line 160):
  **Prohibited evaluative language:** Never use "clean", "correct", "safe", "straightforward", "simple", "trivial",
  or "verbatim" in the summary. These words pre-judge the quality of the change and bias agents toward leniency.

Evidence (line 162):
  **Explicit prohibition:** The summary must never conclude that a refactoring is correct -- that determination is
  the review agents' job.
